OKF MD Master — Complete Guide

Step-by-step walkthrough for scanning, uploading, and managing files with OKF MD.

Login Required — Only Google and GitHub accounts are accepted. No PIN or guest access.

1 — Login with OAuth

1
Choose Google or GitHub to sign in
Click Google → redirects to Google OAuth
[ SCREENSHOT: Login page with Google and GitHub buttons ]
Click Login with Google or Login with GitHub on the top-right of the header. After OAuth approval, your account is created automatically. First-time users require admin activation.

2 — Upload Files

2
Drag or select .md, .json, .csv files
Click +Upload → File dialog opens
[ SCREENSHOT: Upload button + file picker ]
Click the + Upload button in the Client dashboard. Select .md, .txt, .json, .csv, .yaml, or .yml files (max 500 KB each). Uploads go into the admin review queue.

3 — Approval Workflow

3
Admin reviews and approves your file
Admin sees uploads in Approval-Widget
[ SCREENSHOT: Admin approval queue with Approve/Reject buttons ]
The admin reviews your upload. If approved, it appears in your Library. If rejected, the reason appears in your Upload History with the reason code (duplicate, wrong_format, too_large, no_content, policy, other).

4 — Client Dashboard

4
Library, Journal, and Chat tabs
Three tabs: Library · Upload History · Chat
[ SCREENSHOT: Client dashboard with file grid ]
Library: View approved files. Upload History: Full journal of all uploads with status and rejection reasons. Chat: Contact support.

5 — Device Scan Wizard

5
Scan your device and convert to .md
Wizard guides you step-by-step
[ SCREENSHOT: Scan Wizard with steps ]
Visit /scan to use the Device Scan Wizard. It walks you through converting documents, images, and code to structured .md files.

6 — Admin Portal

6
Founder admin at /admin
/admin shows approval queue + user management
[ SCREENSHOT: Admin dashboard with Approval-Widget ]
Admins access the Approval Queue, activate/deactivate users, and manage the system from /admin.
